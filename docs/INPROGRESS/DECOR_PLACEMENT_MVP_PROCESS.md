# Decor Placement MVP Process

## Цель

Сделать визуальное MVP для выставления декораций в комнате:

- игрок видит комнату и точки, куда можно поставить декор;
- в нижней панели видит уже полученные декорации;
- выбирает декорацию из панели и ставит ее в подходящее место;
- может открыть информацию по декорации;
- может кликнуть уже установленную декорацию и снять ее;
- после перезахода установленные декорации отображаются на своих местах.

Для первого MVP достаточно поддержать две декорации: кактус и кофе-машину / кофейник. Покупка сейчас не входит в задачу: предметы можно выдавать через читы.

## Главное решение для MVP

Точки размещения лучше сделать на UI, а не на сцене.

Причина: текущий референс выглядит как полноэкранная 2D-иллюстрация комнаты с UI-точками поверх нее. Для MVP это проще, быстрее и безопаснее по адаптации под устройства. Не нужно заводить камеру, world-space raycast, сортировку 2D/3D объектов и отдельную логику конвертации координат.

Комната должна быть UI-слоем внутри `DecorPlacementWindow`:

- фон комнаты - `Image` / `RawImage` с сохранением пропорций;
- точки декора - UI-кнопки, привязанные к прямоугольнику фона;
- поставленная декорация - UI-изображение внутри той же точки;
- нижний список декораций - обычная UI-панель поверх комнаты.

Важно: точки не должны стоять в абсолютных пикселях экрана. Их координаты должны быть относительно rect'а фона комнаты. Тогда на разных устройствах фон масштабируется, а точки остаются на нужных местах относительно картинки.

## Существующая логика, которую нужно использовать

Источник правды уже есть в домене декораций:

- `DecorPlacementService` хранит и меняет расстановку;
- `BookShopConfig.DecorSlots` описывает доступные слоты;
- `DecorConfig` описывает саму декорацию;
- размещение валидно, если:
  - `DecorConfig.PositionType == DecorSlot.PositionType`;
  - `DecorConfig.Size <= DecorSlot.MaxSize`;
  - слот свободен;
  - декорация есть в инвентаре игрока;
  - эта декорация еще не стоит в другом слоте.

UI не должен хранить отдельное состояние расстановки. UI только показывает состояние сервиса и отправляет команды `PlaceAsync` / `UnplaceAsync`.

Пример настройки:

```json
{
  "id": "cart_table_1",
  "positionType": "Standing",
  "maxSize": "Medium"
}
```

```json
{
  "id": "cactus",
  "displayName": "Cactus",
  "iconAddress": "Decor/Cactus",
  "positionType": "Standing",
  "size": "Small"
}
```

Если кактус должен ставиться на `cart_table_1`, его `positionType` должен быть `Standing`, а `size` не больше `Medium`.

## UX-процесс

### 1. Default

Окно только открыто.

Показываем:

- комнату;
- все доступные точки;
- уже установленные декорации;
- нижний список доступных предметов;
- кнопку закрытия.

Если слот пустой, показываем marker / pin. Если слот занят, показываем декорацию, а marker можно скрыть или сделать менее заметным.

### 2. Выбор декорации из нижнего списка

Игрок кликает карточку декорации, например кактус.

После клика:

- выбранная карточка получает selected-state;
- подсвечиваются только подходящие пустые слоты;
- неподходящие слоты скрываются или становятся disabled;
- повторный клик по выбранной карточке отменяет выбор.

Для MVP не нужен drag and drop. Самый быстрый и понятный сценарий:

1. клик по декорации;
2. подсветка подходящих мест;
3. клик по месту;
4. декорация сразу ставится.

Drag and drop можно оставить на будущее. Он дороже по реализации и хуже дружит с разными устройствами, safe area, скроллом нижнего списка и точными попаданиями пальцем.

### 3. Размещение

Игрок кликает подсвеченную точку.

UI вызывает `DecorPlacementService.PlaceAsync(decorId, slotId)`.

Если команда успешна:

- в слоте появляется визуал декорации;
- проигрывается короткая анимация;
- проигрывается звук размещения, если клип назначен;
- selected-state с карточки снимается;
- подсветка слотов выключается.

Если команда неуспешна, UI просто возвращается в понятное состояние: выбор остается активным или сбрасывается, а ошибку можно залогировать. Для MVP отдельный error popup не нужен.

### 4. Клик по установленной декорации

Игрок кликает уже установленный предмет.

Показываем маленький HUD рядом со слотом:

- `Replace`;
- `Remove`.

На MVP:

- `Remove` работает;
- `Replace` можно показать, но оставить disabled или без действия;
- HUD закрывается при клике вне него, закрытии окна или выборе предмета из нижней панели.

### 5. Remove

Игрок нажимает `Remove`.

UI вызывает `DecorPlacementService.UnplaceAsync(slotId)`.

Если команда успешна:

- установленная декорация исчезает с простой анимацией;
- на слоте снова появляется marker / pin;
- предмет остается в нижнем списке, потому что он не удаляется из инвентаря.

### 6. Info

На карточке декорации в нижней панели есть кнопка `info`.

Клик по ней открывает отдельное overlay-окно / popup поверх текущего UI:

- иконка декорации;
- название;
- бонусы;
- описание;
- одна кнопка закрытия.

Info popup не должен менять выбор декорации и не должен сбрасывать расстановку.

## Состояния окна

Минимальный набор состояний:

```text
Default
DecorSelected
PlacedSlotSelected
InfoPopupOpen
```

`Default` - ничего не выбрано.

`DecorSelected` - выбрана декорация из нижнего списка, подсвечены подходящие слоты.

`PlacedSlotSelected` - выбран уже установленный предмет, показан HUD `Replace / Remove`.

`InfoPopupOpen` - поверх текущего состояния открыта информация по предмету.

Будущее состояние для полноценной замены:

```text
ReplacePreview
```

В нем можно будет повторить flow со скрина: выбрать установленный предмет, нажать `Replace`, выбрать новый предмет, показать preview, затем `Apply` / `Cancel`.

Для текущего MVP `ReplacePreview` не нужен.

## Визуальная анимация

Для установки декорации достаточно простой DoTween-анимации:

- scale `0.85 -> 1.08 -> 1.0`;
- alpha `0 -> 1`;
- длительность примерно `0.2-0.3` секунды.

Для снятия:

- scale `1.0 -> 0.85`;
- alpha `1 -> 0`;
- после завершения очистить view слота и вернуть marker.

Анимация должна быть только визуальной. Состояние в сервисе меняется через `PlaceAsync` / `UnplaceAsync`, а view затем обновляется по результату.

## Звук

Звук размещения лучше подключить как serialized `AudioClip` на view / controller окна.

Когда размещение успешно:

```text
Audio.PlayUi(placeClip)
```

Если клипа пока нет, поле может быть пустым, а вызов должен быть no-op. Логика аудио в проекте уже есть, поэтому отдельную систему под декор заводить не нужно.

Для remove можно позже добавить отдельный клип. Для MVP достаточно звука на place.

## Загрузка после перезахода

При открытии `DecorPlacementWindow` нужно:

1. получить текущие placement'ы через `DecorPlacementService.GetAllPlacements()`;
2. для каждого `(slotId, decorId)` найти UI anchor слота;
3. найти `DecorConfig` для `decorId`;
4. отрисовать иконку / визуал декорации в нужном слоте.

Важный технический риск: `DecorPlacementService` должен быть создан до загрузки save data, потому что он регистрирует save hook. Если сервис создается лениво только при открытии окна, сохраненная расстановка может не загрузиться после перезахода. Для MVP это нужно проверить и при необходимости форсировать создание сервиса в bootstrap / installer до `SaveDataLoadOperation`.

## Настройка prefab'а

`DecorPlacementWindow` нужно переделать из debug/list UI в визуальное окно.

Рекомендуемая структура:

```text
DecorPlacementWindow
  RoomRoot
    RoomBackground
    SlotsRoot
      DecorSlotAnchorView cart_table_1
      DecorSlotAnchorView cart_table_2
      DecorSlotAnchorView ...
  BottomInventoryPanel
    CategoryTabs
    ItemsRoot
  SelectedSlotHud
    ReplaceButton
    RemoveButton
  InfoPopupRoot
```

Каждый `DecorSlotAnchorView` должен иметь:

- `slotId`;
- marker button;
- highlight object;
- placed decor image;
- optional selected outline.

`slotId` обязан совпадать с id из `bookshops.json`. Например:

- `cart_table_1`;
- `cart_table_2`;
- `cart_table_3`;
- `hang_1`;
- `wall_left`.

Для финального MVP можно настроить только два рабочих слота под `Standing` декорации, если кактус и кофе-машина оба имеют `Standing`.

## Что нужно сделать для MVP

1. Добавить / проверить конфиги двух декораций:
   - cactus: `Standing`, `Small`;
   - coffee machine / coffee pot: `Standing`, `Small` или `Medium`.

2. Выдать эти предметы игроку через текущие читы.

3. Переделать `DecorPlacementWindow` в визуальное окно:
   - фон комнаты;
   - UI anchors для слотов;
   - нижний список инвентаря;
   - info button на карточке;
   - selected-slot HUD с `Replace` и `Remove`.

4. Реализовать click-to-place flow:
   - клик по карточке;
   - подсветка совместимых пустых слотов;
   - клик по слоту;
   - `PlaceAsync`;
   - визуальная отрисовка.

5. Реализовать remove flow:
   - клик по установленной декорации;
   - HUD;
   - `Remove`;
   - `UnplaceAsync`;
   - очистка слота.

6. Добавить простую DoTween-анимацию на place/remove.

7. Подключить звук на успешный place через существующую audio-логику.

8. Проверить сохранение:
   - поставить кактус;
   - поставить кофе-машину;
   - закрыть игру;
   - открыть игру;
   - открыть окно декора;
   - оба предмета должны быть на тех же местах.

## Что не входит в MVP

- drag and drop;
- полноценный replace flow с preview и `Apply / Cancel`;
- покупка декораций;
- world-space точки на сцене;
- отдельная сцена декораций;
- сложные эффекты, частицы и уникальные анимации для каждого предмета.

## Рекомендация

Для текущей задачи идти через UI anchors и click-to-place. Это закрывает нужный результат с минимальным риском: две декорации можно поставить, снять, сохранить и восстановить после перезахода. При этом архитектура остается совместимой с текущим `DecorPlacementService`, а будущий replace flow можно добавить поверх тех же slot anchors без переделки базовой модели.
