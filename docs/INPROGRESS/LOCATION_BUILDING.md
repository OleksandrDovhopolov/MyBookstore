# Построение локации

Первый вариант. Документ описывает **как собрать визуальную локацию** (набережная, пляж, городская улица, парк и т.д.) в портретной ориентации под мобильный Android. Геймплейная часть локации (что считается шансы, как тратятся слоты декора) — в [CORE_LOOP.md §2 / §3](../CORE_LOOP.md) и [DECOR.md](../DECOR.md); здесь только про **сборку сцены**.

Аудитория: разработчик + художник. ТЗ для художника собирается из §6 (список ассетов) и §7 (worked example).

---

## 1. Визуальное направление

**Стиль — painterly / watercolor**, как в Tiny Bookshop (референс — [REFERENCE_TinyBookshop_Day.md](../REFERENCE_TinyBookshop_Day.md)). **НЕ пиксель-арт.**

Что это значит на практике:
- Мягкие пастельные тона, без агрессивных контрастов.
- Тени и блики нарисованы **на самих спрайтах**; 2D Lights — только для атмосферы, не как основной источник теней.
- Текстуры спрайтов — высокого разрешения, фильтр **Bilinear** (не Point).
- Полноэкранный шейдер бумажного зерна + лёгкая виньетка усиливают «единый мир из иллюстрации».
- Палитра локации: **10–14 цветов max**, единая по всем локациям проекта.

Что НЕ делаем:
- Не миксуем painterly с пиксель-артом.
- Не используем 3D-модели в портретной сцене (даже если это так в референсе).
- Не закладываем `Pixel Perfect Camera` — он дискретизирует пан и убивает painterly-плавность.

## 2. Базовые технические параметры

| Параметр | Значение | Где задаётся |
|---|---|---|
| **Reference resolution** | `1080 × 2400` | Canvas Scaler + камера |
| **Aspect ratio** | 9:20 (current Android standard) | Дизайн арта |
| **Camera projection** | Orthographic 2D | Main Camera в `GameplayScene` |
| **PPU (Pixels Per Unit)** | `100` | На каждом спрайте при импорте |
| **Camera orthographic size** | `12` (= 2400 / (2 × 100)) | Main Camera |
| **Sprite Filter Mode** | `Bilinear` | Default Sprite Import Settings |
| **Sprite Compression** | `Normal Quality` (или ASTC 6×6 на Android) | Default Sprite Import Settings |
| **Sprite Mipmaps** | `On` (для крупных >256 px) | Per-sprite |
| **Renderer** | URP **2D Renderer** | `UniversalRenderPipelineAsset` |
| **Canvas Scaler** | `Scale With Screen Size`, `Match = 0.5` | Каждый Canvas |
| **2D Lights** | Включены (Global + точечные) | Через URP 2D Renderer |

**Safe area подход:** все критичные элементы (фургон, прилавок, HUD) умещаются в центральные `1080 × 2000`. Sky и floor расширены до полного `1080 × 2400`. На телефонах 9:16…9:21 обрезаются только края sky/floor — ничего важного не теряется.

## 3. Анатомия портретной локации — слои

Локация — это **«вертикальный театр»**. От дальнего к ближнему по `Z` / Sorting Layers:

| # | Sorting Layer | Что внутри | Order |
|---|---|---|---|
| 1 | `BG_Sky` | Небо, облака, далёкие силуэты (горы, далёкая скала с маяком) | 0–9 |
| 2 | `BG_Mid` | Море/горизонт, парапет набережной, дальние здания | 10–19 |
| 3 | `Floor` | Пол сцены (плитка, песок, асфальт), мокрые лужи | 20–29 |
| 4 | `Decor_Back` | Высокие декорации (фонари, флагштоки, столбы, билборды) | 30–39 |
| 5 | `Subject` | Главный объект локации — фургон-лавка, открытый прилавок и т.д. | 40–49 |
| 6 | `Decor_Front` | Низкие декорации (стулья, корзины, цветы, ящики) | 50–59 |
| 7 | `Characters` | Покупатели, питомцы, чайки и прочая живность | 60–69 |
| 8 | `FG_Frame` | Передний план: бортик, клумба, низкая стенка — отрезают игрока | 70–79 |
| 9 | `FX_Weather` | Дождь, снег, лепестки, листья — поверх всего, кроме UI | 80–89 |
| 10 | `UI` | HUD, окна (отдельный Canvas) | через Canvas |

Post-process (paper grain + vignette) накладывается **после** всех слоёв сцены, **до** UI Canvas — Renderer Feature `AfterRenderingTransparents`.

## 4. Структура префаба локации

Каждая локация — **один Addressable префаб**, адрес лежит в `LocationConfig.PrefabAddress` (см. [Configs.Models.LocationConfig](../../Assets/Game/Features/Configs/Models/LocationConfig.cs)).

Шаблон иерархии:

```
Location_<Name> (root GameObject, на нём LocationController)
├── 01_BG_Sky
│   ├── Sky                  SpriteRenderer, layer=BG_Sky, order=0
│   ├── Clouds_Far           SpriteRenderer, layer=BG_Sky, order=1
│   └── Silhouette_Far       SpriteRenderer, layer=BG_Sky, order=2  (горы, скалы)
├── 02_BG_Mid
│   ├── Horizon              SpriteRenderer, layer=BG_Mid, order=0
│   ├── Sea_Or_City          SpriteRenderer, layer=BG_Mid, order=1
│   └── PromenadeWall        SpriteRenderer, layer=BG_Mid, order=2
├── 03_Floor
│   └── FloorPlate           SpriteRenderer, layer=Floor, order=0
├── 04_Decor_Back
│   ├── StreetLamp           SpriteRenderer + IInteractable? (false)
│   ├── FlagPole             SpriteRenderer
│   └── Sign                 SpriteRenderer
├── 05_Subject
│   ├── ShopBody             SpriteRenderer + SalesShelf (центральный объект)
│   ├── ShopAwning           SpriteRenderer
│   ├── ShopWindow_Glow      SpriteRenderer (отдельный для смены день/ночь)
│   └── ShelfAnchors         empty GameObjects: ShelfSlot_01..N
├── 06_Decor_Front
│   └── DecorSlots           empty GameObjects: DecorSlot_01..N
│       ├── DecorSlot_01     (Transform — куда инстанцируется выбранный декор из инвентаря)
│       └── DecorSlot_02
├── 07_Characters
│   └── NpcSpawnPoints       empty GameObjects:
│       ├── Spawn_Entry      (точка появления покупателя)
│       ├── ApproachShelf    (куда подойти к полке)
│       ├── ApproachCounter  (куда подойти к прилавку)
│       └── Spawn_Exit       (точка ухода)
├── 08_FG_Frame
│   └── CurbOrFence          SpriteRenderer, layer=FG_Frame, order=0
├── 09_FX_Weather
│   └── (вкладывается рантайм-системой погоды по WeatherId из MorningContext)
├── CameraAnchor             empty Transform — куда центрировать камеру при загрузке
├── CameraBounds             PolygonCollider2D — границы пана (для будущего Cinemachine Confiner)
└── Lighting                 (опц.) Global Light 2D + точечные Spot Light 2D
    ├── GlobalLight_Day      Light2D, тип Global
    ├── GlobalLight_Night    Light2D, тип Global, по умолчанию выключен
    └── LampGlow             Light2D, Point, дочерний к StreetLamp / ShopWindow
```

**Правила именования:**
- Контейнеры слоёв — `01_BG_Sky`, `02_BG_Mid` и т.д. с цифровым префиксом. Сразу видна вертикальная иерархия в Hierarchy.
- Слот-якоря — `<Type>Slot_<NN>`: `DecorSlot_01`, `ShelfSlot_03`. Двузначный индекс, чтобы сортировались правильно.
- NPC spawn — `Spawn_<Purpose>`: `Spawn_Entry`, `Spawn_Exit`.

## 5. Слот-система

Якоря — пустые `Transform`-объекты, которые сервисы заполняют рантайм:

| Тип слота | Кто заполняет | Содержание |
|---|---|---|
| `ShelfSlot_NN` | `IShelfBookSpawner` (или `SalesShelf` сам) | Спрайт книги из `BookConfig.ShelfSprite` |
| `DecorSlot_NN` | `IDecorPlacementService` (см. [DECOR.md](../DECOR.md)) | Префаб декора из инвентаря игрока |
| `Spawn_*` | `ICustomerSpawner` (см. [BookSell](../../Assets/Game/Features/BookSell/Services/ICustomerSpawner.cs)) | Префаб покупателя |
| `WallAnchor_*` | `IDecorPlacementService` (опц. для настенного декора) | Постеры, часы, картины |

**Контракт слота:**
- Имя по конвенции выше.
- Локальная позиция — это **центр привязки спрайта** (не пивот объекта-родителя).
- Поворот — обычно `0,0,0`. Если декор должен быть наклонён — поворот выставляется здесь, а не на спрайте.
- Масштаб — `1,1,1`. Никаких скейлов в слотах, иначе масштабируются все размещённые объекты.

**Чтение слотов LocationController'ом** (на старте, разово):
```csharp
// псевдокод
var shelfSlots = transform.Find("05_Subject/ShelfAnchors")
                  .Cast<Transform>().OrderBy(t => t.name).ToArray();
var decorSlots = transform.Find("06_Decor_Front/DecorSlots")
                  .Cast<Transform>().OrderBy(t => t.name).ToArray();
// и т.д. — публикуются через ILocationContext
```

### Универсальные (шарятся между всеми локациями, рисуются один раз)

| Ассет | Размер (1×) | Заметки |
|---|---|---|
| `seagull_idle.png` | 120×100 | Чайка, 2 ракурса |

### Освещение для этой локации (пасмурно + дождь)
- `GlobalLight_Day` — Intensity 0.8, цвет холодно-серый (`#9AA8B5`).
- `LampGlow` (точечный, прикреплён к `StreetLamp` и `caravan_window_glow`) — Intensity 1.2, тёплый (`#F4C879`), радиус 3 units. Включён, потому что пасмурно.
- `GlobalLight_Night` — выключен.

### Weather FX
- `09_FX_Weather` рантайм заполняется системой погоды из `MorningContext.WeatherId == "rain_light"`.
- Префаб дождя — partikkel system со скроллом текстуры `rain_tex.png` по экранному UV. Альфа 0.5. Spawn по всему вью камеры.

## 8. Композиционные правила (do / don't)

### DO
- Дальний план **светлее**, ближний **насыщеннее** — главный приём для глубины (не размер).
- Сильный вертикальный якорь сверху (фонарь, флагшток, дерево) — иначе верхняя половина экрана пустая.
- Foreground frame в нижних 10–15% — «отрезает» игрока от сцены, даёт ощущение присутствия.
- Главный субъект (фургон) — центр кадра, ~40–50% ширины экрана.
- Один точечный 2D Light на тёплый источник (лампа, окно лавки) — фокус внимания.

### DON'T
- Не миксуй пиксель-арт и painterly спрайты в одной сцене.
- Не клади важные элементы в полосы 0–80 px и 2320–2400 px (могут обрезаться на узких 9:16 телефонах).
- Не делай 6+ уровней parallax-параллакса — на портрете не работает, лучше 3 (sky / mid / floor).
- Не вешай Collider2D на декоративные спрайты — только на интерактивные.
- Не используй `Pixel Perfect Camera` — она ломает painterly.
- Не размещай UI поверх сильно детализированных областей фона — потеряется контраст. Резервируй верхние 200 px и нижние 240 px под HUD.

## 10. Регламент создания новой локации

### Шаг 1 — Композиция (без художника, плейсхолдеры)
1. Скопировать `Assets/Game/Features/Location/Prefabs/Location_Template.prefab` (создать один раз) → `Location_<Name>.prefab`.
2. Заполнить все слои **цветными прямоугольниками** (Sprites/Square из URP) в цвета палитры будущей сцены.
3. Расставить слот-якоря (`DecorSlot_*`, `ShelfSlot_*`, `Spawn_*`).
4. Запустить на телефоне (Build & Run или Unity Remote), проверить:
   - Композиция читается.
   - HUD не перекрывает субъект.
   - Все слоты на правильных позициях.

### Шаг 2 — ТЗ художнику
1. Скриншот сцены с плейсхолдерами + размеры экрана.
2. Палитра 10–14 цветов (можно взять с скриншота через color picker).
3. Список ассетов по §6 — конкретные имена файлов + размеры в px.
4. 2–3 референс-скриншота из Tiny Bookshop / Cozy Grove / Spiritfarer.
5. Срок: 5–7 дней на локацию.

### Шаг 3 — Интеграция готового арта
1. Спрайты складываются в `Assets/Art/Locations/<Name>/`.
2. Sprite Atlas: `Atlas_Location_<Name>.spriteatlas`, добавить всю папку.
3. Спрайты заменяют плейсхолдеры в префабе **один к одному** (поэтому размеры в ТЗ важны).
4. Префаб маркируется Addressable, адрес — `location/<name>`.
5. В `locations.json` прописать `prefabAddress` = `location/<name>`.
6. Smoke-тест: запустить с этой локацией как выбранной в Preparation.

### Шаг 4 — Релиз
1. `Tools/Configs/Sync Bundled Defaults to StreamingAssets` — чтобы новые `locations.json` лежали в bundled defaults.
2. Build → проверить на физическом девайсе.

## 11. Каталог референсных локаций (Tiny Bookshop)

Шесть локаций-кандидатов для портирования. Сложность оценена по: количество уникальных PNG, плотность декора, сложность освещения, переиспользование общих ассетов.

**Шкала сложности:** 🟢 LOW · 🟡 MED · 🔴 HIGH.

### 11.1 Promenade / Набережная — 🟡 MED
*(Полный worked example — см. §7)*

- **Палитра:** холодная серо-голубая, пасмурно.
- **Якорь:** флагшток слева + фонарь по центру.
- **Уник. ассетов:** ~20.
- **Особенность:** дождь как FX, единый плоский задник (море), bg сильно упрощается.

### 11.2 Lighthouse / Маяк — 🟢 LOW

**Композиция:** прибрежный холм с маяком и небольшим домиком. Осенняя растительность (жёлтая трава, рыжие кусты), телеграфные столбы с проводами, овечки на переднем плане, ограждение-забор, пикап (опц.). Лёгкий дождь.

**Почему 🟢:** структура естественно вертикальная (маяк = идеальный портретный якорь). Растительность из 2–3 тайлящихся куст-спрайтов покрывает половину канваса дёшево. Background = sea horizon + простой силуэт скалы. Цветовая палитра узкая (осенние охры + холодное небо). FX дождя переиспользуется с Promenade.

**Слои:**
| Слой | Содержание |
|---|---|
| `BG_Sky` | Облака + далёкий силуэт (соседняя скала). |
| `BG_Mid` | Маяк + домик смотрителя + море. |
| `Floor` | Грунтовая дорога с лужами. |
| `Decor_Back` | Уличные фонари (×2), телеграфные столбы (×2), провода. |
| `Subject` | Фургон-лавка (шаренный с другими локациями). |
| `Decor_Front` | Кусты осенние (тайлинг), валуны, скамейка, забор из досок. |
| `Characters` | Овцы (новый ассет, ~120×100). |
| `FG_Frame` | Большие валуны слева снизу. |
| `FX_Weather` | Дождь (шаренный). |

**Уникальных ассетов:** ~18 (из них 6 переиспользуется с Promenade: каравана, дождь, фонарь, общие декор-слоты).
**Что новое нужно нарисовать:** маяк (силуэт), домик смотрителя, осенние кусты × 2, телеграфные столбы × 2, валуны, забор из досок, овца.

**Освещение:** `GlobalLight_Day` холодный (`#A8B5C2`, intensity 0.85). Точечный `Light2D` на верхушке маяка (тёплый импульс — анимация intensity 0.6→1.0).

---

### 11.3 Beach / Пляж — 🟡 MED

**Композиция:** песчаный пляж с фургоном в дюнах. Скала с маяком слева на дальнем плане, море по горизонту, травянистые дюны справа, фонарь, пикап. Покупатели в пляжной одежде. Солнечный день.

**Почему 🟡:** требует разделения BG на 3 зоны (скала / море / дюны) — каждая со своим силуэтом. Песок занимает много экрана — нужно крупное полотно `floor_sand.png` без видимых стыков. Зато освещение простое (ясный день, без сложных теней) и FX вообще нет.

**Слои:**
| Слой | Содержание |
|---|---|
| `BG_Sky` | Голубое небо с пушистыми облаками + дальний силуэт скалы с маяком (слева). |
| `BG_Mid` | Море с белой полосой прибоя, отдельный силуэт обрыва справа с травой. |
| `Floor` | Песок + узкая полоса грунтовой дороги. |
| `Decor_Back` | Уличный фонарь (правая сторона), ограждение набережной. |
| `Subject` | Фургон-лавка с пляжным декором (гамак сбоку, флажки гирлянды, спасательный круг). |
| `Decor_Front` | Складной стул, столик, ящик, пляжный зонт, ракета-игрушка, цветочные горшки. |
| `Characters` | Покупатели в пляжных образах (шорты, шляпы) — отдельные кастомы. |
| `FG_Frame` | Передний песчаный бугор / куст травы. |

**Уникальных ассетов:** ~22.
**Что новое:** скала-силуэт другого ракурса (отличается от Lighthouse), пляжная трава дюн, зонт-палатка (на дальнем плане слева), гамак-сетка, спасательный круг (можно повесить на фургон).

**Освещение:** `GlobalLight_Day` тёплый (`#FFEFC2`, intensity 1.0). Без точечных. Опционально — лёгкий blur shader на горизонте морe для «жары».

### 11.4 Supermarket / Mega Marché — 🟢 LOW

**Композиция:** парковка перед супермаркетом. Большое красное здание с вывеской, праздничные гирлянды-флажки, уличные фонари, тележки и велосипеды на переднем плане, прохожие. Ясный день.

**Почему 🟢:** background — **одно** большое здание-фасад без слоистых деталей. Геометрия чисто архитектурная (прямоугольники), без природных текстур. Цветовая палитра 6–8 цветов. Меньше всего уникальных декор-объектов из всех 6 локаций.

**Слои:**
| Слой | Содержание |
|---|---|
| `BG_Sky` | Голубое небо с лёгкими облаками + дальние холмы (силуэт). |
| `BG_Mid` | Фасад супермаркета (одно крупное полотно с вывеской «MÉGA MARCHÉ»). |
| `Floor` | Серый асфальт парковки + бордюр. |
| `Decor_Back` | Уличные фонари (×2), гирлянда-флажки между ними, мусорные баки у входа. |
| `Subject` | Фургон-лавка (шаренный). |
| `Decor_Front` | Гидрант, ящики, цветочные тумбы. |
| `Characters` | Покупатели (семья, ребёнок) + общая толпа на заднем плане. |
| `FG_Frame` | Тележки супермаркета + велосипеды на переднем плане (левый-нижний и правый-нижний углы). |

**Уникальных ассетов:** ~14.
**Что новое:** фасад супермаркета (1 большой sprite ~1200×900), вывеска «BOOKS» в стилистике супермаркета, тележка ×1, велосипед ×1, гирлянда-флажки (тайлинг).

**Освещение:** `GlobalLight_Day` нейтральный (`#FFFFFF`, intensity 1.0). Без точечных.

**Минус:** визуально наименее «уютная» из шести — может выпадать из общей атмосферы. Подходит как контраст («городской выезд» в кампании дней).

---

### 11.5 Flea Market / Блошиный рынок — 🔴 HIGH

**Композиция:** внутренний двор средневековой деревни с рынком. Несколько тентов разных цветов (жёлтый, красный, розовый, зелёный), художник с картинами слева, арка из камня, многоэтажные старинные здания и башня замка вдалеке, деревья. Ясный день, тёплый.

**Почему 🔴:** очень высокая плотность уникальных объектов. **5+ тентов** разных форм. Несколько мини-сцен внутри сцены (художник + рамы с картинами — это отдельный «магазинчик»). Архитектура заднего плана — каменные дома с черепичными крышами + башня с часами. Деревья крупные, не тайлятся.

**Слои:**
| Слой | Содержание |
|---|---|
| `BG_Sky` | Розовато-голубое небо, тёплые облака. |
| `BG_Mid` | Средневековые дома (несколько уникальных силуэтов, разные крыши), башня замка справа, крупное дерево слева. |
| `Floor` | Бежевая мощёная плитка + грунт. |
| `Decor_Back` | Каменная арка по центру дальнего плана, фонарь, столб. |
| `Subject` | Фургон-лавка (шаренный). |
| `Decor_Front` | Тенты × 5 (разной формы и цвета), стойки с картинами, чемоданы, табуретки, складная палатка соседнего торговца, картины на стойках. |
| `Characters` | Покупатели + художник + другие торговцы (несколько уникальных силуэтов). |
| `FG_Frame` | Низкая ограда / бордюр. |

**Уникальных ассетов:** ~38.
**Что новое:** 5 тентов, 6+ картин на стойках, каменная арка, старинная башня с часами, дом-силуэт ×2–3, чемоданы (×2 вариант), складная стойка, художник-NPC + другие торговцы.

**Освещение:** `GlobalLight_Day` тёплый (`#FFE4B5`, intensity 1.05). Без точечных.

**Решение:** оставить на последнюю итерацию. Хорошая локация, но не первая.

### 11.6 Cafe Liberté — 🔴 HIGH (вечерняя)

**Композиция:** старая европейская улица вечером. Синее здание кафе с вывеской «Café Liberté», ставни с цветами, плющ/виноград на фасадах, гирлянды флажков-вымпелов, многочисленные подвесные фонари, цветочные клумбы на переднем плане, фургон-лавка по центру. Замок/большой дом на дальнем хребте. Тёплое вечернее освещение.

**Почему 🔴:** **сложнейшая для освещения** локация. 10+ горящих фонарей и подвесных ламп, каждая нуждается в точечном `Light2D`. Архитектура заднего плана многоярусная (нижний этаж + верхний этаж + дальний замок). Виноградные лозы — детализированный органический декор. Вечерняя палитра требует двух глобальных лайтов (`Night` + `Lamps`) и точной настройки blend mode.

**Слои:**
| Слой | Содержание |
|---|---|
| `BG_Sky` | Вечернее розово-фиолетовое небо. |
| `BG_Mid` | Дальний замок/большой дом, многоярусные средневековые здания. |
| `Floor` | Мощёная плитка тёмного оттенка (мокрая после дождя?). |
| `Decor_Back` | 8+ настенных фонарей (горящих) + 2 уличных фонаря, гирлянды флажков, подвесные лампы. |
| `Subject` | Фургон-лавка. |
| `Decor_Front` | Цветочные клумбы × 3–4 (большие, в горшках), кафе-столики и стулья (часть кафе), зонт-навес у кафе, фургон. |
| `Characters` | Прохожие (опц., вечером можно меньше людей). |
| `FG_Frame` | Большие клумбы в нижних углах. |

**Уникальных ассетов:** ~40.
**Что новое:** фасад кафе с вывеской, виноградные лозы (тайлящаяся текстура), 3+ типа настенных фонарей, гирлянда вымпелов, многоуровневая архитектура (нижний фасад + верхний этаж + дальний замок), цветочные клумбы крупные, кафе-столики (часть набора).

**Освещение** (самое затратное):
- `GlobalLight_Night` (`#5566AA`, intensity 0.45) — вечерняя база.
- Каждая лампа: `Light2D` Point, тёплый (`#FFC880`), радиус 1.5 unit, intensity 0.8 — **минимум 10 точек света**. Это влетит в Light Cull cost на мобиле.
- Один большой `Light2D` Sprite Light (с текстурой светового пятна) на окно фургона.

**Бюджет рендера:** на средне-бюджетном Android (Snapdragon 6-серии) с 10+ Light2D — может уйти в просадку 30 fps. **Перед сборкой — профайлинг.** Опция оптимизации: «запечь» свечение фонарей прямо в спрайты задника (нарисовать жёлтое сияние), а Light2D оставить только на 2–3 ключевых.

**Решение:** прекрасная локация для финального WOW-эффекта, но **не для первой итерации**. Сборка после того, как освещён хотя бы один светлый день-сценарий и есть бюджетный baseline.

### Что собирать первым — рекомендация

**Promenade** (§7) — уже есть worked example, прорабатывался в обсуждении. Идёт первой.

**Lighthouse** — второй. **Самая выгодная по соотношению «новизна / затраты»:**
1. ✅ Максимум переиспользования с Promenade (фургон, дождь, фонарь, общий шаблон префаба).
2. ✅ Естественная портретная композиция (маяк — вертикальный якорь).
3. ✅ Узкая палитра, лёгкая для художника.
4. ✅ Растительность тайлится — много канваса за дёшево.
5. ✅ Освещение простое (день + один импульс на маяке).
6. ✅ Визуально сильно отличается от Promenade (тёплая осень vs холодная пасмурность) — игрок чувствует «новое место», хотя половина ассетов шарится.

**Supermarket** — третьей, если нужна локация-контраст («поехали в город»). Минимум ассетов, но менее уютная.

**Beach** — четвёртой. Чуть сложнее по background (3 зоны), но даёт солнечную локацию-курорт (важно для разнообразия настроения).

**Flea Market** — пятой. Когда уже отлажен пайплайн декора и есть готовый набор NPC-силуэтов.

**Cafe Liberté** — последней. Тяжёлая по освещению + вечерняя палитра. Прекрасно смотрится как «WOW-локация» в маркетинговых скриншотах, но требует уже отстроенного бюджета рендера.

**Минимальный набор для MVP:** Promenade + Lighthouse + одна солнечная (Supermarket или Beach). Этого хватит, чтобы игрок проходил 5–7 дней без визуального переутомления.

## 12. Генерация ассетов — workflow и промт-шаблоны

Документ описывает **процесс** работы с image-gen моделями (Gemini Imagen / Flux / DALL-E 3) для создания спрайтов и фоновых слоёв. Сама генерация выполняется во внешних инструментах — здесь только дисциплина и шаблоны.

### 12.1 Решение по пайплайну: Hybrid

Рекомендованный гибрид для MVP:

| Что | Источник | Почему |
|---|---|---|
| Sky, Sea, далёкие силуэты (cliff, lighthouse, башни) | **AI gen** | Дёшево, плохо опознаётся как AI |
| Foliage, кусты, трава, камни | **AI gen** | Тайлинг + повторяемость; стиль drift не критичен |
| Декор-пропсы (ящики, фонари, стулья, цветы) | **AI gen** | На периферии внимания |
| Уличные пол/floor plates | **AI gen** | Большие плоские текстуры — модель отдаёт стабильно |
| **Hero subject** (фургон-лавка) | **Художник** | В центре внимания каждой локации — style drift убьёт «уют» |
| **Покупатели / NPC** | **Художник** | Анимация + узнаваемые персонажи — AI даёт несогласованную пластику |
| **UI иконки, корешки книг** | **Художник** | Мелкие, требуют чёткости |

Если бюджета на художника нет совсем — full-AI допустим, но с принятием style drift и disclosure-требований (см. §12.5).

### 12.2 Style Reference — главный артефакт пайплайна

**До любой генерации ассетов** делается **одна composite-картинка** локации в финальном стиле — это `style_reference.png`. Все последующие генерации обязаны иметь её как input image + промт-якорь `in the style of the attached reference image`.

Хранение:
```
Assets/Art/Reference/
├── TinyBookshop/                       — скрины оригинала (минимум 10)
│   ├── ref_promenade_rain.png
│   ├── ref_lighthouse_autumn.png
│   └── ...
├── StyleReference/
│   ├── style_reference_v1.png          — финальный composite, утверждённый
│   └── style_reference_v1_prompt.md    — точный промт, которым он сгенерирован
└── Palettes/
    └── palette_global.png              — 10–14 цветов проекта (color picker source)
```

`StyleReference/` версионируется в git (это маленькие файлы). Когда стиль меняется — новая версия (`_v2`), старая остаётся.

### 12.6 Quality control

После каждой пачки в 3–4 ассета:
1. Закинуть в Unity в существующий префаб-плейсхолдер.
2. Запустить на телефоне рядом с уже принятыми ассетами.
3. Проверка по чек-листу:
   - [ ] Палитра не уплывает (использовать color picker по 5–7 ключевым цветам в `palette_global.png`).
   - [ ] Толщина «нарисованных» обводок consistent с reference.
   - [ ] Перспектива совпадает (3/4 top-down).
   - [ ] Нет AI-артефактов (лишних пальцев у NPC, нечитаемых букв на вывесках, кривой геометрии).
   - [ ] Edges чистые: если magenta-key — фон удалён, fringe убран. Если transparent PNG — alpha-канал ровный.
4. Если хоть один пункт не сошёлся — **перегенерация**, не «починим в Photoshop». Иначе технический долг накапливается.

### 12.7 Юридическая часть (disclosure)

**При коммерческом релизе на Google Play / App Store:**
- Google Play Console → Content Rating → секция **AI Generated Content** — declare.
- App Store → Privacy Section — пока explicit AI declaration не требуется, но политика меняется ежемесячно — проверять перед каждым релизом.
- В **EULA / Privacy Policy** игры — упомянуть использование AI-сгенерированного контента и какая модель использовалась (Gemini Imagen / Flux / etc.).
- **Хранить промты + reference images** к каждому ассету (`Assets/Art/Reference/StyleReference/`) — на случай audit или копирайт-спора.

При full-human или hybrid (где AI — только background) disclosure часто **не требуется** для второстепенных elements — но это решение юриста, не разработчика. Если сомневаешься — declare.

### 12.8 Хранение reference-картинок и промтов

Каждая принятая генерация хранится с метаданными:

```
Assets/Art/Locations/Promenade/
├── bg_sky_overcast.png                  — финальный ассет
├── bg_sky_overcast.png.meta             — Unity import settings
├── _source/
│   ├── bg_sky_overcast_prompt.md        — промт, версия модели, дата
│   └── bg_sky_overcast_raw.png          — исходник до edge cleanup
```

`_source/` НЕ попадает в build (через `.gitignore` или Editor-only assembly), но **остаётся в git** для traceability.
