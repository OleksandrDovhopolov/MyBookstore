# Утро

## Назначение

`Утро` открывает игровой день и задаёт контекст для выбора игрока в `Подготовке`: что сегодня происходит в городе, какая погода, какие события активны, какие локации выглядят выгоднее, и есть ли сюжетный/персонажный крючок на день.

Для MVP это короткий экран перед подготовкой, а не отдельная большая сцена. Его задача - быстро дать игроку причину осознанно выбрать локацию, книги и декор.

## Анализ старта Tiny Bookshop

На старте референсной игры игрок не получает сложную стратегическую панель. Его вводят в роль начинающего владельца маленькой мобильной лавки: он приезжает в Bookstonbury, знакомится с локальной жительницей Tilde, получает мягкий туториал и начинает с ограниченным набором книг/места/декора. Первые игровые дни работают как обучение ритму: поставить лавку, подобрать stock, открыть продажи, получить деньги и вложить их обратно в книги/декор.

Ключевые выводы для нас:

- Утро должно быть коротким и атмосферным, чтобы не тормозить 3-6 минутную сессию.
- Первый день должен объяснять цель через персонажа/событие, а не через сухую инструкцию.
- Информация утром должна влиять на решение в подготовке: погода, событие, аудитория локации, активный запрос персонажа.
- С самого начала нужно показать игроку, что день - это не рандомный набор клиентов, а маленькая ситуация в городе.

## MVP-поведение

Минимальный экран утра:

1. Показать номер дня.
2. Показать короткую сводку дня: `weather`, `news/event`, `hint`.
3. Показать 1 активный модификатор спроса.
4. Показать 1 мягкую цель дня или сюжетную заметку.
5. Кнопка перехода в `Подготовку`.

Для первого прототипа достаточно 1 локации, но экран уже должен поддерживать несколько локаций, чтобы не переписывать flow в Sprint 3.

## Игровой поток

```text
Load day state
  -> Resolve active event/weather/news
  -> Build morning summary
  -> Show morning screen
  -> Player taps Continue
  -> Open Preparation
```

## Входные данные

| Источник | Данные | MVP |
|---|---|---|
| Save | текущий день, золото, открытые локации, прогресс персонажей | да |
| EventConfig | событие дня, модификатор спроса, текст сводки | да |
| LocationConfig | список доступных локаций и их аудитории | частично |
| RequestConfig | специальные запросы дня | позже |

## Выходные данные

| Данные | Куда передаются |
|---|---|
| `activeDayId` | все этапы дня |
| `activeEventId` | Подготовка, Продажа, Итоги |
| `weatherId` | Подготовка, Продажа |
| `demandModifiers` | Подготовка, Продажа |
| `dailyGoalId` | Итоги |

## Настройки

Для MVP достаточно расширить `EventConfig` или добавить отдельный `DayConfig`.

Минимальные поля:

```json
{
  "id": "day_001",
  "dayIndex": 1,
  "title": "Первый день у парка",
  "weatherId": "clear",
  "eventId": "exam_week",
  "summaryText": "В университете началась неделя экзаменов.",
  "hintText": "Студенты чаще ищут научпоп и короткие книги.",
  "demandGenreBonus": ["science", "classic"],
  "targetLocationIds": ["university"]
}
```

## UI

Экран должен быть читаемым за 5-10 секунд:

- `День N`
- 1-2 строки новости/события
- погода как иконка + короткий текст
- карточка активного бонуса спроса
- CTA `К подготовке`

Не добавлять длинные диалоги утром в MVP. Если нужен персонажный ввод, он должен быть коротким: 1 реплика + кнопка.

## Analytics

Минимальные события:

| Event | Params |
|---|---|
| `morning_opened` | `day`, `event_id`, `weather_id` |
| `morning_continue_clicked` | `day`, `read_time_sec` |

## Save

После входа на экран утра день считается начатым, но не завершённым.

Сохранять:

- `currentDay`
- `currentPhase = morning`
- `activeEventId`
- `weatherId`

## Acceptance Criteria

- Игрок видит экран утра перед подготовкой.
- Сводка дня загружается из конфигов или стабильного fallback.
- Переход в `Подготовку` работает без перезапуска сцены.
- При перезапуске игры на фазе утра восстанавливается тот же день и тот же event/weather.
- Нет хардкода конкретного дня в UI-контроллере.

## Out Of Scope

- Полная календарная система.
- Несколько параллельных событий.
- Сложные диалоги утром.
- Генеративные новости.
- Персонажные цепочки, кроме 1 короткого вводного хука.
